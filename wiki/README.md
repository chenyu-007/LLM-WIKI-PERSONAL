# wiki

`wiki/` 是可维护知识层。这里的页面不是原始资料，而是经过整理、拆分、链接后的 Markdown 知识。

## 分类

- `concepts/`：概念页。回答“这个术语、方法、模式是什么”。例如 `Embedding`、`RAG Pipeline`。
- `entities/`：实体页。记录“这个具体对象是谁/是什么”。例如人物、组织、产品、项目、论文、工具。
- `sources/`：来源总结页。回答“这一份原始资料说了什么”。必须能追溯到 `raw/`。
- `syntheses/`：综合页。回答“多个来源合起来说明什么”。例如学习地图、主题综述、路线图、横向对比。
- `questions/`：问题页。回答“这个反复出现的问题该怎么判断”。例如“RAG 和 Fine-tuning 怎么选”。

## 快速归类

新增页面前先问：

1. 单一资料摘要？放 `sources/`。
2. 具体对象？放 `entities/`。
3. 抽象概念？放 `concepts/`。
4. 多来源综合？放 `syntheses/`。
5. 可复用问题？放 `questions/`。

详细边界见 [schema](../schema/README.md)。

## 写作原则

- 一个页面只处理一个主题。
- 每个重要结论都尽量指向 `raw/` 来源。
- 页面之间要互相链接，不要形成孤岛。
- 不确定内容放进 `待核实`，不要写成确定事实。
